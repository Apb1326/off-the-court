# League calibration targets (Stage 1)

Generated by `npx tsx scripts/derive-league-targets.ts` from the normalized
NBA data contracts in `data/nba/normalized/` (Stage 0 pipeline output).
**Do not hand-edit this file** — re-run the script (it is a pure,
deterministic function of the normalized data; `--check` verifies the
committed copy byte-for-byte). Derived values are hand-transcribed, with
provenance annotations, into `scripts/profile-engine.ts` (targets) and
tuned toward in `src/engine/constants.ts` (base knobs are NOT direct
transcriptions — see the targets-vs-base-constants note below).

- Season range: 2023-24, 2024-25, 2025-26 (default = last 3 completed seasons; modern-game window is a design decision)
- Normalized schema_version: 3; nba_api versions in manifest: 1.11.4
- Games: 3690 (2023-24: 1230, 2024-25: 1230, 2025-26: 1230); team-games: 7380
- Shot attempts (shot_events): 657387; heaves excluded from zone tables: 1721 (2023-24: 701, 2024-25: 836, 2025-26: 184)
- Made-FG assist join coverage: 100.000% (309532/309532; gate ≥ 99.9%)

All rates are computed from summed counts across the pooled sample — never
averaged across players or seasons. Tolerance band per target =
max(max single-season |value − pooled|, documented floor); the floors are
judgment calls (see `FLOORS` in the script) sized above the profile's
sim sampling noise at ~1,290 games.

## Tier assignment (fixed before tuning; never outcome-based)

**ENFORCED (profile pass/fail):** pace, pts, ppp, fga, fg%, 3pa, 3p%, fta,
ft%, oreb, dreb, reb, ast, stl, blk, tov, avg margin; realized FG% per
engine zone (6); six-zone FGA shares; three-bucket rim/mid/three shares.

**INFORMATIONAL (logged, never fails):** play-type distribution (owner:
Stage 2 — selection mechanism is hardcoded in `src/engine/play-types.ts`,
out of Stage 1 scope), assisted-shot rate by zone (owner: Stage 2/3 — the
NBA scorekeeper assist definition is materially more liberal than the
engine's strict pass-into-the-make chain), PBP transition cross-check
(owner: Stage 2), turnover-type mix / and-one rate / putback rate /
offensive-rebound rate (owner: Stage 3).

## Six-zone mapping (settled Stage 1 decision)

NBA shot-chart zones → engine zones (heave exclusion applied first):

| Engine zone | Rule |
|---|---|
| rim | Restricted Area |
| short_midrange | In The Paint (Non-RA), or Mid-Range < 14 ft |
| long_midrange | Mid-Range ≥ 14 ft |
| corner_three | Left Corner 3 + Right Corner 3 |
| above_break_three | Above the Break 3 < 27 ft |
| deep_three | Above the Break 3 ≥ 27 ft |

Judgment calls: Paint (Non-RA) belongs to `short_midrange` (median distance
7 ft — floater/short-roll territory, matching the engine's semantic intent;
`rim` stays at-basket finishes only). The NBA's single Mid-Range zone is
split at 14 ft (standard short/long midrange convention). `deep_three`
is defined as above-the-break threes at ≥ 27 ft — below the 32 ft
heave-distance cutoff; it is the fastest-trending zone in the pooled window.
This resolves the Mid-Range/deep-three split flagged for Stage 1 review in
`pipeline/lib/zones.py` (the resolution lives here + in `constants.ts`;
the Python provisional five-zone mapping is unchanged).

Heave rule (explicit, not a blanket time cut): excluded iff NBA zone is
"Backcourt" OR (shot distance ≥ 32 ft AND ≤ 3 s left in the period).
`shotDistance` is integer feet (floor-truncated), so the field comparisons
are exact true-distance cuts. Heaves stay in league FGA/FG%/3PA/3P%
(box-score consistency); they are excluded from per-zone FG% and shot mix.

## Targets vs base constants (binding rule)

Observed league values below are **profile targets** (the pass/fail oracle
for realized engine output). They are never transcribed directly into
`BASE_FG_PCT_BY_ZONE` or other base constants: realized FG% = base + the
average of the modifier stack (contest, shooter/defender, fatigue, play
type, form, spacing), which does not average to zero. Base knobs are tuned
until realized profile output lands on target.

Sources & formulas (numerator / denominator; all "per team-game" use
team-games = 7380):

- `fga`, `fg%`, `3pa`, `3p%` — shot_events (all attempts, heaves included;
  3PT = `shotType === "3PT Field Goal"`). FG% = ΣFGM/ΣFGA.
- `fta`, `ft%` — pbp `Free Throw` rows; made = description without the
  `MISS ` prefix (pbp FT rows carry `shotResult: null`). Cross-checked
  against box_advanced Σ(fta×gp) within 0.5%.
- `tov` — pbp `Turnover` rows (includes team turnovers, matching the real
  box convention and the sim's possession-ending turnover universe).
- `oreb`,`dreb`,`reb`,`ast`,`stl`,`blk` — box_advanced Σ(perGame×gp) over
  players (player-credited universe, matching how the sim accumulates
  team totals from player events). perGame values are rounded to 1
  decimal upstream; the reconstruction error is random-sign and small
  relative to the tolerance floors.
- `pts`, `avg abs margin` — games contract, from `participants[].score`
  (10 games across 2024-25/2025-26 have null home/away designations but
  full participant scores).
- `pace` — poss/team-game = FGA + 0.44·FTA + TOV − ORB (identical to the
  sim-side estimate in scripts/profile-engine.ts, so both sides share one
  definition; the NBA's own possession convention would disagree by a
  formula constant). `ppp` = pts/poss.

## Caveats

- Pooling 2023-24..2025-26 averages across a live trend (3PA and the deep-
  three share rose across the window), so pooled targets sit slightly
  behind the newest season by design; the season-deviation tolerance term
  absorbs this.
- 2025-26 heave-rule basis shift: from 2025-26 the NBA charges end-of-
  period heaves as team attempts, removing them from player shot charts —
  hence the season's much lower excluded-heave count above. For 2025-26
  the "heaves stay in league FGA/FG%/3PA/3P%" convention therefore only
  partially applies (the rule-removed ~0.25 FGA/team-game never reach
  shot_events, flattering that season's 3P% by ~+0.003). The effect is
  well inside every tolerance band; a future single-season 2025-26+
  window should re-decide the convention deliberately.
- The putback FGA share above is the immediate tip/putback ATTEMPT proxy,
  not Synergy's putback POSSESSION concept (~5-6% of possessions incl.
  kick-outs and fouls drawn off offensive rebounds). Stage 3 must not
  tune the engine putback play-type frequency to this number.
- box_advanced perGame values are rounded upstream (1 decimal); the
  Σ(perGame×gp) reconstruction carries small random-sign error.
- pbp free-throw made/missed is inferred from the `MISS ` description
  prefix (the contract carries `shotResult: null` for FTs).
- Synergy OffRebound (Putbacks) is not harvested; see play-type section.
- Jensen caveat: centered or re-centered engine inputs do not guarantee
  the aggregate lands where arithmetic says (nonlinear mix shifts,
  clamping). The profile run is the only proof a retune is done.
